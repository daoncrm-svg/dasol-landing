이 워크플로우는 코딩 용어를 모르는 비개발자 사용자가 일상 언어로 아이디어를 던졌을 때, Product Manager(기획자) 역할을 맡아 가장 적합한 기술 스택을 선정하고 최종 개발 지시 프롬프트를 만들어주는 과정입니다.

## 1단계: 요구사항 분석 (Requirement Analysis)
사용자가 겪고 있는 문제나 만들고자 하는 서비스를 분석합니다.
- 사용자의 일상 언어를 기술적 요구사항으로 번역합니다.
- 해당 아이디어를 구현하기 가장 좋은 최신 기술 스택(프론트엔드, 백엔드, DB 등)을 결정합니다.
- 현재 프로젝트의 코드베이스를 분석하여 기존 구조와 패턴을 파악합니다.

## 2단계: 글로벌 전문가 스킬 색출 (Best Skill Discovery) — 반드시 수행할 것!
> **필수 참조 파일:** `C:\Users\ksj86\.gemini\antigravity\knowledge\awesome-agent-skills\README.md`

이 단계를 절대 건너뛰지 마세요. 반드시 위 경로의 README.md 파일을 view_file 도구로 읽고, 현재 문제와 가장 관련 있는 전문가 스킬(팀/카테고리)을 최소 2~3개 찾아서 사용자에게 추천해야 합니다.

수행 절차:
1. 위 README.md 파일을 view_file로 열어서 전체 카탈로그를 스캔합니다.
2. 사용자의 요구사항과 매칭되는 스킬 카테고리 및 구체적 스킬을 식별합니다.
3. 찾아낸 스킬의 이름, 출처(팀), 깃허브 링크를 사용자에게 보여줍니다.
4. 해당 스킬 저장소에서 참고할 수 있는 패턴이나 베스트 프랙티스를 요약합니다.

## 3단계: 개발 에이전트를 위한 프롬프트 도출 (Prompt Generation)
2단계에서 색출한 전문가 스킬 정보와 1단계의 기술 분석을 결합하여, 사용자가 그대로 복사-붙여넣기 할 수 있는 '전문적이고 구체적인 개발 지시 명령어'를 생성합니다.

출력 형식:
```
📋 개발 지시 프롬프트
/agent-team-builder [구체적이고 실행 가능한 개발 지시 내용]
### 참고 스킬:
- [스킬명] by [팀명]: [깃허브 URL]
### 수정 사항:
1. [구체적 파일명과 변경 내용]
2. ...
```

## 4단계: 비개발자를 위한 친절한 용어 정리 (Glossary)
이번 작업에서 등장한 기술 용어들을 일반인도 이해하기 쉬운 비유를 섞어 3줄 이내로 깔끔하게 정리합니다.
- 예: "CORS (코스): 아파트 경비실이 등록된 차량만 들여보내는 보안 명단"
- 예: "API: 식당에서 손님(화면)이 주방(서버)에 주문을 전달하는 웨이터"
